<Info>
NFT retrieval allows you to access detailed information about non-fungible tokens including metadata, images, attributes, and ownership details across multiple blockchains.
</Info>

## Overview

NFT retrieval is essential for:
- **Portfolio management**: Viewing complete NFT collections across wallets
- **Marketplace integration**: Displaying NFTs for trading and showcasing
- **Game development**: Accessing in-game assets and collectibles
- **Art galleries**: Showcasing digital art collections and exhibitions

<Tip>
Use filtering options like `includePossibleSpam` and `includeNsfw` to customize the NFT results based on your application's requirements.
</Tip>

## Prerequisites

To retrieve NFTs, you need:
- Valid wallet IDs, addresses, or contract information
- Understanding of NFT standards (ERC-721, ERC-1155)
- Knowledge of supported blockchain networks
- No authentication required for basic NFT queries

## Method 1: Get NFTs by Wallet ID

Retrieve all NFTs owned by a specific Venly-managed wallet.

### API Reference

#### Endpoint
```http
GET /api/wallets/{walletId}/nonfungibles
```

#### Parameters

| Parameter | Type | Description | Required | Example |
|-----------|------|-------------|----------|---------|
| `walletId` | Path | The unique ID of the wallet | ✅ | `"da8d6b3b-5e4a-4e9d-b12f-59d68b396d6f"` |
| `includePossibleSpam` | Query | Include potential spam NFTs | ❌ | `false` |
| `includeNsfw` | Query | Include NSFW (adult) content | ❌ | `false` |

### Code Examples

<CodeGroup>

```http Get All NFTs (Clean Results)
GET /api/wallets/da8d6b3b-5e4a-4e9d-b12f-59d68b396d6f/nonfungibles?includePossibleSpam=false&includeNsfw=false
```

```http Get All NFTs (Include Everything)
GET /api/wallets/da8d6b3b-5e4a-4e9d-b12f-59d68b396d6f/nonfungibles
```

```http Get NFTs with Spam Filter
GET /api/wallets/f21dc1e5-4253-487e-8927-3b5247437f99/nonfungibles?includePossibleSpam=false
```

</CodeGroup>

## Method 2: Get NFTs by Address and Blockchain

| `includePossibleSpam` | Query | Include potential spam NFTs | ❌ | `false` |
| `includeNsfw` | Query | Include NSFW content | ❌ | `false` |

## Content Filtering Options

<Accordion title="Understanding Content Filters">

**Spam Filtering (`includePossibleSpam`)**
- `false` (recommended): Excludes suspected spam NFTs
- `true`: Includes all NFTs, including potential spam
- Default: `true` (includes everything)

**NSFW Filtering (`includeNsfw`)**
- `false`: Excludes adult/NSFW content
- `true`: Includes all content regardless of rating
- Default: `true` (includes everything)

**Best Practices:**
- Use `includePossibleSpam=false` for cleaner user experiences
- Use `includeNsfw=false` for family-friendly applications
- Combine both filters for the cleanest results

</Accordion>

## NFT Standards Support

<Accordion title="Supported NFT Standards">

**ERC-721 (Ethereum)**
- Unique, non-fungible tokens
- Each token has a unique ID
- One owner per token

**ERC-1155 (Multi-Token)**
- Semi-fungible tokens
- Multiple copies of the same token possible
- Batch operations supported
- `balance` field indicates quantity owned

**Cross-Chain Support:**
- Ethereum mainnet
- Polygon (MATIC)
- Binance Smart Chain (BSC)
- Arbitrum
- Avalanche (AVAX)
- And more supported networks

</Accordion>

## Integration Examples

### NFT Gallery Component
```javascript
async function loadNFTGallery(walletId, options = {}) {
  const params = new URLSearchParams();
  
  if (options.excludeSpam) {
    params.append('includePossibleSpam', 'false');
  }
  
  if (options.familyFriendly) {
    params.append('includeNsfw', 'false');
  }
  
  const response = await fetch(
    `/api/wallets/${walletId}/nonfungibles?${params}`
  );
  const data = await response.json();
  
  if (data.success) {
    return data.result.map(nft => ({
      id: nft.id,
      name: nft.name,
      image: nft.imageThumbnailUrl || nft.imageUrl,
      contract: nft.contract.address,
      attributes: nft.attributes
    }));
  }
  
  throw new Error('Failed to load NFTs');
}
```
