<Info>
Wallet retrieval allows you to access detailed information about wallets including balances, metadata, and configuration settings.
</Info>

## Overview

Retrieving wallet information is essential for:
- **Wallet management**: Accessing wallet details and configurations
- **Balance monitoring**: Checking current wallet balances and status
- **User portfolio**: Viewing all wallets belonging to a specific user
- **Application logic**: Making decisions based on wallet properties

<Tip>
Use specific query methods to efficiently retrieve exactly the wallet information you need without unnecessary data transfer.
</Tip>

## Prerequisites

To retrieve wallet information, you need:
- Valid wallet IDs, user IDs, or identifiers
- Appropriate permissions to access the requested wallets
- No authentication required for basic wallet information

## Method 1: Get Wallet by ID

Retrieve detailed information for a specific wallet.

### API Reference

#### Endpoint
```http
GET /api/wallets/{id}
```

#### Parameters

| Parameter | Type | Description | Required | Example |
|-----------|------|-------------|----------|---------|
| `id` | Path | The unique wallet ID | ✅ | `"96d7f016-a062-4d29-9e37-94b9765bc08a"` |

## Response Fields

| Field | Description |
|-------|-------------|
| `id` | Unique wallet identifier |
| `address` | Blockchain address of the wallet |
| `walletType` | Type of wallet (API_WALLET, etc.) |
| `secretType` | Blockchain network (ETHEREUM, MATIC, etc.) |
| `createdAt` | Wallet creation timestamp |
| `archived` | Whether the wallet is archived |
| `description` | Human-readable wallet description |
| `primary` | Whether this is the user's primary wallet |
| `hasCustomPin` | Whether wallet has a custom PIN |
| `identifier` | Custom identifier assigned to wallet |
| `userId` | ID of the wallet owner |
| `custodial` | Whether wallet is custodial |
| `balance` | Current balance information |
| `balance.exchange` | USD price and value information |

## Pagination Best Practices

<Warning>
When retrieving large numbers of wallets, always use pagination to avoid performance issues and timeouts.
</Warning>

### Recommended Pagination Settings
- **Default page size**: 20-50 items
- **Maximum page size**: 100 items
- **Always check**: `hasNextPage` before requesting more data

### Pagination Example
```javascript
async function getAllUserWallets(userId) {
  let allWallets = [];
  let page = 0;
  let hasMore = true;
  
  while (hasMore) {
    const response = await fetch(
      `/api/wallets?userId=${userId}&page=${page}&size=50`
    );
    const data = await response.json();
    
    allWallets.push(...data.result);
    hasMore = data.pagination.hasNextPage;
    page++;
  }
  
  return allWallets;
}
```

## Best Practices

1. **Use specific queries** to reduce data transfer
2. **Implement pagination** for large result sets
3. **Cache wallet data** appropriately to reduce API calls
4. **Filter by user ID** when managing user portfolios
5. **Use identifiers** for logical wallet grouping
6. **Monitor balance changes** for important wallets
