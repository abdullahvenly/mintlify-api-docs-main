<Info>
Wallet import allows you to bring existing wallets into the Venly platform, giving you unified management across multiple blockchain networks.
</Info>

## Overview

Wallet import is essential for:
- **Existing wallet integration**: Bringing wallets from other platforms
- **Multi-chain management**: Using the same wallet across compatible blockchains
- **Backup recovery**: Restoring wallets from private keys or keystores
- **Platform migration**: Moving wallets from other services to Venly

<Tip>
Always verify wallet addresses after import to ensure the correct wallet was imported successfully.
</Tip>

## Prerequisites

Before importing a wallet, ensure you have:
- The wallet's private key, keystore file, or WIF format
- A valid user ID to link the wallet to
- The user's signing method credentials for authentication
- Knowledge of the source blockchain network

## Import Methods

<Steps>
<Step title="Private Key Import">
Import using the wallet's private key (most common method).
</Step>

<Step title="Keystore Import">
Import using an encrypted keystore file with password.
</Step>

<Step title="WIF Import">
Import Bitcoin/Litecoin wallets using Wallet Import Format.
</Step>

<Step title="Cross-Chain Migration">
Migrate existing Venly wallets to compatible blockchains.
</Step>
</Steps>

## API Reference

### Endpoint
```http
POST /api/wallets/import
```

### Headers

| Header | Format | Description | Example |
|--------|--------|-------------|---------|
| `Signing-Method` | `id:value` | Authentication using user's signing method | `756ae7a7-3713-43ee-9936-0dff50306488:123456` |

<Warning>
The signing method must belong to the user specified in the request body. Ensure proper authentication before importing wallets.
</Warning>

## Method 1: Import with Private Key

The most common import method for Ethereum-compatible blockchains.

### Parameters

| Parameter | Type | Description | Required | Example |
|-----------|------|-------------|----------|---------|
| `importWalletType` | String | Import type for the blockchain | ✅ | `"ETHEREUM_PRIVATE_KEY"` |
| `privateKey` | String | The wallet's private key | ✅ | `"0x4c0883a69102937d6231471b5dbb6204fe5129617082792ae468d01a3f362318"` |
| `userId` | String | ID of the user to link wallet to | ❌ | `"aee7260f-9348-4b73-b06b-f62c0f9751e4"` |

### Code Examples

<CodeGroup>

```json Import Ethereum Wallet
{
  "importWalletType": "ETHEREUM_PRIVATE_KEY",
  "privateKey": "0x4c0883a69102937d6231471b5dbb6204fe5129617082792ae468d01a3f362318",
  "userId": "aee7260f-9348-4b73-b06b-f62c0f9751e4"
}
```

```json Import Polygon Wallet
{
  "importWalletType": "MATIC_PRIVATE_KEY",
  "privateKey": "0x7d8a90c8f2b3e4d5a6b7c8d9e0f1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d7e8f9",
  "userId": "b297ff04-26cd-661d-0840-def3e9f3759b"

```


## Method 2: Import with Keystore

Import encrypted keystore files with password protection.

### Parameters

| Parameter | Type | Description | Required | Example |
|-----------|------|-------------|----------|---------|
| `importWalletType` | String | Import type for the blockchain | ✅ | `"ETHEREUM_KEYSTORE"` |
| `keystore` | String | JSON keystore as escaped string | ✅ | See example below |
| `password` | String | Keystore password | ✅ | `"mySecurePassword123"` |
| `userId` | String | ID of the user to link wallet to | ❌ | `"aee7260f-9348-4b73-b06b-f62c0f9751e4"` |

## Method 3: Import with WIF

Import Bitcoin and Litecoin wallets using Wallet Import Format.

### Parameters

| Parameter | Type | Description | Required | Example |
|-----------|------|-------------|----------|---------|
| `importWalletType` | String | WIF import type | ✅ | `"BITCOIN_WIF"` |
| `wif` | String | Wallet Import Format string | ✅ | `"5HueCGU8rMjxEXxiPuD5BDku4MkFqeZyd4dZ1jvhTVqvbTLvyTJ"` |
| `passphrase` | String | WIF passphrase (if required) | ❌ | `"myPassphrase"` |
| `userId` | String | ID of the user to link wallet to | ❌ | `"aee7260f-9348-4b73-b06b-f62c0f9751e4"` |

### Code Examples

<CodeGroup>

```json Import Bitcoin Wallet
{
  "importWalletType": "BITCOIN_WIF",
  "wif": "5HueCGU8rMjxEXxiPuD5BDku4MkFqeZyd4dZ1jvhTVqvbTLvyTJ",
  "userId": "aee7260f-9348-4b73-b06b-f62c0f9751e4"
}
```

```json Import Litecoin with Passphrase
{
  "importWalletType": "LITECOIN_WIF_PASSPHRASE",
  "wif": "6PfQu77ygVyuUxZjuuMp9RGy85LQ8672AFoqxsbmgxwXwxkKbWS5K5cNd",
  "passphrase": "mySecurePassphrase123",
  "userId": "b297ff04-26cd-661d-0840-def3e9f3759b"
}
```

</CodeGroup>

## Method 4: Cross-Chain Migration

Migrate existing Venly wallets to compatible blockchains using the same private key.

### Supported Migrations

| From | To | Use Case |
|------|----|---------| 
| ETHEREUM | MATIC | Use Ethereum wallet on Polygon |
| MATIC | ETHEREUM | Use Polygon wallet on Ethereum |
| ETHEREUM | BSC | Use Ethereum wallet on BSC |
| BSC | ETHEREUM | Use BSC wallet on Ethereum |

### Parameters

| Parameter | Type | Description | Required | Example |
|-----------|------|-------------|----------|---------|
| `importWalletType` | String | Must be `"MIGRATION"` | ✅ | `"MIGRATION"` |
| `walletId` | String | ID of existing Venly wallet | ✅ | `"a076dd02-04ab-439b-8618-626a4b44f99f"` |
| `to` | String | Target blockchain | ✅ | `"MATIC"` |
| `userId` | String | ID of the user to link wallet to | ❌ | `"aee7260f-9348-4b73-b06b-f62c0f9751e4"` |

### Code Examples

<CodeGroup>

```json Migrate Ethereum to Polygon
{
  "importWalletType": "MIGRATION",
  "walletId": "a076dd02-04ab-439b-8618-626a4b44f99f",
  "to": "MATIC",
  "userId": "aee7260f-9348-4b73-b06b-f62c0f9751e4"
}
```

```json Migrate BSC to Ethereum
{
  "importWalletType": "MIGRATION",
  "walletId": "b186ee03-15bc-550c-9729-cdf2d8f2648a",
  "to": "ETHEREUM",
  "userId": "b297ff04-26cd-661d-0840-def3e9f3759b"
}
```

</CodeGroup>

## Supported Import Types

<Accordion title="Complete List of Import Types">

**Ethereum-Compatible Chains:**
- `ETHEREUM_PRIVATE_KEY`, `ETHEREUM_KEYSTORE`
- `MATIC_PRIVATE_KEY`, `MATIC_KEYSTORE`
- `BSC_PRIVATE_KEY`, `BSC_KEYSTORE`
- `ARBITRUM_PRIVATE_KEY`, `ARBITRUM_KEYSTORE`
- `AVAC_PRIVATE_KEY`, `AVAC_KEYSTORE`

**Bitcoin-Based Chains:**
- `BITCOIN_WIF`, `BITCOIN_WIF_PASSPHRASE`
- `LITECOIN_WIF`, `LITECOIN_WIF_PASSPHRASE`

**Other Blockchains:**
- `HEDERA_PRIVATE_KEY`, `HEDERA_KEYSTORE`
- `VECHAIN_PRIVATE_KEY`, `VECHAIN_KEYSTORE`
- `NEO_PRIVATE_KEY`, `NEO_KEYSTORE`, `NEO_WIF`
- `IMX_PRIVATE_KEY`, `IMX_KEYSTORE`
- `AETERNITY_PRIVATE_KEY`, `AETERNITY_KEYSTORE`
- `GOCHAIN_PRIVATE_KEY`, `GOCHAIN_KEYSTORE`

**Special Types:**
- `MIGRATION` - Cross-chain wallet migration

</Accordion>

## Error Handling

**Invalid Private Key**
```json
{
  "success": false,
  "errorCode": "INVALID_PRIVATE_KEY",
  "message": "The provided private key is invalid or malformed"
}
```

**Incorrect Keystore Password**
```json
{
  "success": false,
  "errorCode": "INVALID_KEYSTORE_PASSWORD",
```

```json
{
  "success": false,
  "errorCode": "UNSUPPORTED_IMPORT_TYPE",
  "message": "The specified import type is not supported"
}
```

**Wallet Already Exists**
```json
{
  "success": false,
  "errorCode": "WALLET_ALREADY_EXISTS",
  "message": "A wallet with this address already exists"
}
```

</Accordion>

## Security Best Practices

<Warning>
**Security Considerations:**
- Never share private keys or keystores
- Use secure connections (HTTPS) when importing
- Verify wallet addresses after import
- Store backup credentials securely
- Consider using hardware wallets for high-value assets
</Warning>

## Best Practices

1. **Verify addresses** after import to ensure correctness
2. **Test with small amounts** before using imported wallets
3. **Keep backups** of original private keys/keystores
4. **Use strong passwords** for keystore files
5. **Import gradually** rather than bulk importing
6. **Monitor balances** after import completion
