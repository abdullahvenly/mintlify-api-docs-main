<Info>
Gas fees are the cost of executing transactions on blockchain networks. Understanding and optimizing gas fees is crucial for cost-effective blockchain operations.
</Info>

## Overview

Gas fee management is essential for:
- **Cost optimization**: Choosing the right fee level for your transaction priority
- **Transaction timing**: Deciding when to submit transactions based on network conditions
- **Budget planning**: Estimating costs for batch operations or regular transactions
- **User experience**: Providing accurate fee estimates to users

<Tip>
Gas prices fluctuate based on network congestion. Check current fees before submitting important transactions to avoid overpaying or delays.
</Tip>

## Prerequisites

To retrieve gas fees, you need:
- The blockchain network you want to query
- No authentication required (public network data)
- Understanding of gas price units (wei, gwei)

## Gas Fee Concepts

<Steps>
<Step title="Gas Price">
The price per unit of gas, typically measured in gwei (1 gwei = 10^9 wei).
</Step>

<Step title="Gas Limit">
The maximum amount of gas a transaction can consume.
</Step>

<Step title="Transaction Cost">
Total cost = Gas Price × Gas Limit (paid in native tokens).
</Step>

<Step title="Fee Tiers">
Different speed/cost options: slow (cheap), standard, fast (expensive).
</Step>
</Steps>

### Response Fields

| Field | Description |
|-------|-------------|
| `gasPrice` | Gas price in wei (smallest unit) |
| `defaultPrice` | Whether this is the recommended default price |

## Fee Tier Interpretation

The response typically contains 4 fee tiers:

<Accordion title="Fee Tier Breakdown">

**Slow (Economy) - Index 0**
- Lowest gas price
- Longer confirmation time (5-30 minutes)
- Best for non-urgent transactions

**Standard (Recommended) - Index 1**
- `defaultPrice: true`
- Balanced speed and cost
- Typical confirmation time (2-5 minutes)

**Fast - Index 2**
- Higher gas price
- Faster confirmation (1-2 minutes)
- Good for time-sensitive transactions

**Fastest - Index 3**
- Highest gas price
- Quickest confirmation (30 seconds - 1 minute)
- Use for urgent transactions only

</Accordion>

## Gas Price Conversion

<Warning>
Gas prices are returned in wei. Use these conversions for better understanding:
- **1 ETH** = 10^18 wei
- **1 gwei** = 10^9 wei
- **1 gwei** = 0.000000001 ETH
</Warning>

### Conversion Examples

```javascript
// Convert wei to gwei
const gasPriceWei = 21450000000;
const gasPriceGwei = gasPriceWei / 1000000000; // 21.45 gwei

// Calculate transaction cost
const gasLimit = 21000; // Standard transfer
const totalCostWei = gasPriceWei * gasLimit; // 450,450,000,000,000 wei
const totalCostEth = totalCostWei / 1000000000000000000; // ~0.00045 ETH
```

### Common Gas Limits

|------------------|-------------------|

### Cost Calculation Example

```javascript
// Example: ERC-20 transfer on Ethereum
const gasPrice = 21450000000; // wei (21.45 gwei)
const gasLimit = 65000; // ERC-20 transfer
const totalCost = gasPrice * gasLimit; // 1,394,250,000,000,000 wei
const costInEth = totalCost / 1000000000000000000; // ~0.00139 ETH
```

## Best Practices

1. **Check fees before transactions**: Always get current prices
2. **Choose appropriate tier**: Balance cost vs. speed based on urgency
3. **Monitor network conditions**: Avoid peak congestion times when possible
4. **Set reasonable limits**: Don't always use the fastest option
5. **Cache responsibly**: Don't fetch fees too frequently (every 30-60 seconds max)
6. **Handle fluctuations**: Gas prices can change rapidly during high activity

## Integration Examples

### Fee Selection Logic
```javascript
async function selectOptimalGasPrice(secretType, urgency = 'standard') {
  const response = await fetch(`/api/transactions/${secretType}/fees`);
  const data = await response.json();
  
  if (!data.success) {
    throw new Error('Failed to fetch gas prices');
  }
  
  const fees = data.result;
  
  switch (urgency) {
    case 'slow':
      return fees[0].gasPrice;
    case 'standard':
      return fees.find(f => f.defaultPrice)?.gasPrice || fees[1].gasPrice;
    case 'fast':
      return fees[2].gasPrice;
    case 'fastest':
      return fees[3].gasPrice;
    default:
      return fees.find(f => f.defaultPrice)?.gasPrice;
  }
}
```

## Network-Specific Considerations

### Ethereum
- High gas prices during network congestion
- Consider Layer 2 solutions for frequent transactions
- Gas prices vary significantly throughout the day

### Polygon
- Generally lower and more stable gas prices
- Less congestion than Ethereum mainnet
- Faster block times mean quicker confirmations

### BSC
- Moderate gas prices
- Good balance of speed and cost
- Popular for DeFi applications
