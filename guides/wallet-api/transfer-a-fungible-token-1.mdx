<Info>
Fungible token transfers allow you to send ERC-1155 tokens that can have multiple copies, such as in-game currencies, utility tokens, or semi-fungible assets.
</Info>

## Overview

Fungible token transfers are essential for:
- **Gaming economies**: Transferring in-game currencies and resources
- **Utility tokens**: Moving platform tokens between users
- **Reward systems**: Distributing fungible rewards and points
- **Multi-token contracts**: Managing tokens within ERC-1155 collections

<Tip>
ERC-1155 tokens can be both fungible (multiple copies) and non-fungible (unique items) within the same contract. Use this endpoint for transferring fungible tokens with quantities greater than 1.
</Tip>

## Prerequisites

Before transferring fungible tokens, ensure you have:
- A wallet containing the fungible tokens you want to transfer
- The recipient's wallet address or email
- Sufficient native tokens for gas fees
- The wallet owner's signing method credentials for authentication

## Token Transfer Process

<Steps>

## API Reference

### Endpoint
```http
POST /api/transactions/execute
```

### Headers

| Header | Format | Description | Example |
|--------|--------|-------------|---------|
| `Signing-Method` | `id:value` | Authentication using wallet owner's signing method | `756ae7a7-3713-43ee-9936-0dff50306488:123456` |

### Parameters

| Parameter | Type | Description | Required | Example |
|-----------|------|-------------|----------|---------|
| `transactionRequest` | Body | Container for all transaction details | ✅ | See examples below |
| `transactionRequest.type` | Body | Must be `"NFT_TRANSFER"` | ✅ | `"NFT_TRANSFER"` |
| `transactionRequest.walletId` | Body | ID of the wallet sending the tokens | ✅ | `"c4c97f87-8e00-4f3a-8647-2f24ac9b73cb"` |
| `transactionRequest.to` | Body | Recipient wallet address or email | ✅ | `"0x2D24b9DE4F963d1Cfac1a65b6F1a85945d3e92F5"` |
| `transactionRequest.secretType` | Body | Blockchain network | ✅ | `"MATIC"` |
| `transactionRequest.tokenAddress` | Body | Contract address of the token | ✅ | `"0xd687ed37ed8ed56ab91387755432a3c8f197455d"` |
| `transactionRequest.tokenId` | Body | ID of the specific token within the contract | ✅ | `34` |
| `transactionRequest.value` | Body | Number of tokens to transfer | ✅ | `2` |

<Warning>
The deprecated `pincode` parameter should not be used. Always use the `Signing-Method` header for authentication.
</Warning>

## Token Standards and Types

<Accordion title="Understanding ERC-1155 Tokens">

**Fungible vs Non-Fungible:**
- **Fungible tokens**: Multiple identical copies can exist (value > 1)
- **Non-fungible tokens**: Unique items with only one copy (value = 1)
- **Semi-fungible**: Can start as fungible and become non-fungible

**Common Use Cases:**
- **Gaming**: In-game currencies, resources, consumables
- **Collectibles**: Trading cards with multiple editions
- **Utilities**: Platform tokens, access passes, vouchers
- **Rewards**: Points, badges, achievement tokens

**Token ID Structure:**
- Each token type has a unique ID within the contract
- Multiple copies of the same token ID can exist
- Balance represents how many copies a wallet owns

**Contract Examples:**
- Gaming platforms: Different token IDs for gold, gems, potions
- Trading cards: Token IDs for different card types
- Event tickets: Token IDs for different seating categories

</Accordion>

## Integration Examples

### Gaming Platform Integration

```javascript
async function transferGameCurrency(fromWalletId, toAddress, currencyTokenId, amount) {
  // First check if wallet has sufficient balance
  const balance = await checkTokenBalance(fromWalletId, currencyTokenId);
  
  if (balance < amount) {
    throw new Error('Insufficient currency balance');
  }
  
  const transferData = {
    transactionRequest: {
      type: 'NFT_TRANSFER',
      walletId: fromWalletId,
      to: toAddress,
      value: amount,
      secretType: 'MATIC',
      tokenAddress: process.env.GAME_TOKEN_CONTRACT,
      tokenId: currencyTokenId
    }
  };
  
  const response = await fetch('/api/transactions/execute', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Signing-Method': `${signingMethodId}:${signingMethodValue}`
    },
    body: JSON.stringify(transferData)
  });
  
  const result = await response.json();
  
  if (result.success) {
    console.log(`Transfer initiated: ${result.result.transactionHash}`);
    return result.result;
  } else {
    throw new Error(`Transfer failed: ${result.message}`);
  }
}
```

### Batch Transfer System

```javascript
async function batchTransferTokens(transfers) {
  const results = [];
  
  for (const transfer of transfers) {
    try {
      const result = await transferFungibleToken(
        transfer.walletId,
        transfer.recipient,
        transfer.tokenId,
        transfer.amount
      );
      
      results.push({
        recipient: transfer.recipient,
        success: true,
        transactionHash: result.transactionHash
      });
    } catch (error) {
      results.push({
        recipient: transfer.recipient,
        success: false,
        error: error.message
      });
    }
    
    // Add delay between transfers to avoid rate limiting
    await new Promise(resolve => setTimeout(resolve, 1000));
  }
  
  return results;
}
```
