<Info>
User retrieval provides comprehensive information about user accounts, including their signing methods, creation details, and current status.
</Info>

## Overview

Retrieving user information is essential for:
- **Account management**: Accessing user details and configurations
- **Authentication status**: Checking available signing methods
- **Security monitoring**: Reviewing authentication attempts and lockouts
- **User support**: Troubleshooting account-related issues

<Tip>
User information includes all signing methods associated with the account, making it easy to understand the user's authentication options.
</Tip>

## Prerequisites

To retrieve user information, you need:
- A valid user ID from a previously created user
- Appropriate permissions to access the user data
- No authentication required for basic user information

### Response Fields

| Field | Description |
|-------|-------------|
| `result.id` | Unique user identifier |
| `result.reference` | Custom reference assigned during creation |
| `result.createdAt` | User account creation timestamp |
| `result.signingMethods` | Array of all signing methods for this user |

### Signing Method Fields

| Field | Description |
|-------|-------------|
| `id` | Unique signing method identifier |
| `type` | Method type (PIN, EMERGENCY_CODE, BIOMETRIC, PASSWORD) |
| `incorrectAttempts` | Current count of failed authentication attempts |
| `remainingAttempts` | Attempts left before method is locked |
| `lastUsedSuccess` | Timestamp of last successful authentication |
| `hasMasterSecret` | Whether method has access to master secrets |

## Signing Method Status Analysis

<Accordion title="Understanding Signing Method Status">

**Attempt Tracking:**
- `incorrectAttempts`: Increments with each failed authentication
- `remainingAttempts`: Decreases with each failed attempt
- When `remainingAttempts` reaches 0, the method is temporarily locked

**Master Secret Access:**
- `hasMasterSecret: true`: Method can access wallet private keys
- `hasMasterSecret: false`: Method has limited access (e.g., view-only)

**Usage Patterns:**
- `lastUsedSuccess`: Helps identify active vs. inactive methods
- Missing `lastUsedSuccess`: Method has never been used successfully

**Security Implications:**
- High `incorrectAttempts`: Possible brute force attempts
- Low `remainingAttempts`: Method approaching lockout
- No recent `lastUsedSuccess`: Potentially compromised or forgotten method

</Accordion>

## Security Monitoring

Use the retrieved information to monitor user security:

### Authentication Health Check
```javascript
function assessUserSecurity(userData) {
  const issues = [];
  
  userData.signingMethods.forEach(method => {
    // Check for high failed attempts
    if (method.incorrectAttempts >= 5) {
      issues.push(`${method.type} has ${method.incorrectAttempts} failed attempts`);
    }
    
    // Check for low remaining attempts
    if (method.remainingAttempts <= 2) {
      issues.push(`${method.type} is close to lockout (${method.remainingAttempts} attempts left)`);
    }
    
    // Check for inactive methods
    const lastUsed = new Date(method.lastUsedSuccess);
    const daysSinceUse = (Date.now() - lastUsed.getTime()) / (1000 * 60 * 60 * 24);
    if (daysSinceUse > 90) {
      issues.push(`${method.type} hasn't been used in ${Math.floor(daysSinceUse)} days`);
    }
  });
  
  return issues;
}
```

## User Management Insights

### Signing Method Analysis
- **PIN**: Primary authentication method, should always be present
- **EMERGENCY_CODE**: Backup method for account recovery
- **BIOMETRIC**: Convenience method for mobile applications
- **PASSWORD**: Additional security layer for sensitive operations

### Best Practices for User Data
1. **Regular monitoring**: Check authentication patterns for anomalies
2. **Proactive support**: Contact users with locked methods
3. **Security alerts**: Notify users of suspicious activity
4. **Method diversity**: Encourage multiple signing methods
5. **Usage tracking**: Monitor method effectiveness and adoption

### User Dashboard Data
```javascript
async function getUserDashboardData(userId) {
  const response = await fetch(`/api/users/${userId}`);
  const userData = await response.json();
  
  if (!userData.success) {
    throw new Error('Failed to fetch user data');
  }
  
  const user = userData.result;
  
  return {
    userId: user.id,
    reference: user.reference,
    accountAge: calculateAccountAge(user.createdAt),
    signingMethods: user.signingMethods.map(method => ({
      type: method.type,
      status: method.remainingAttempts > 0 ? 'active' : 'locked',
      lastUsed: method.lastUsedSuccess,
      security: method.hasMasterSecret ? 'full' : 'limited'
    })),
    securityScore: calculateSecurityScore(user.signingMethods)
  };
}
```
