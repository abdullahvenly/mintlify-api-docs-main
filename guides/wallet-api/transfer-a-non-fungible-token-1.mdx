<Info>
NFT transfers allow you to send unique digital assets like collectibles, art, gaming items, and other non-fungible tokens between wallets.
</Info>

## Overview

NFT transfers are essential for:
- **Digital collectibles**: Trading unique art, cards, and memorabilia
- **Gaming assets**: Transferring characters, weapons, and in-game items
- **Marketplace transactions**: Buying and selling NFTs on platforms
- **Gifting and rewards**: Sending unique tokens as gifts or achievements

<Tip>
This endpoint works for both ERC-721 (unique tokens) and ERC-1155 (multi-token standard) NFTs. For ERC-1155, this transfers exactly one copy of the specified token.
</Tip>

## Prerequisites

Before transferring NFTs, ensure you have:
- A wallet containing the NFT you want to transfer
- The recipient's wallet address or email
- Sufficient native tokens for gas fees
- The wallet owner's signing method credentials for authentication

## NFT Transfer Process

<Steps>
<Step title="Verify NFT Ownership">
Confirm that the sender wallet owns the specific NFT to be transferred.
</Step>

<Step title="Prepare Transfer Details">
Gather the NFT contract address, token ID, and recipient information.
</Step>

<Step title="Set Recipient Address">
Specify the destination wallet address or email address.
</Step>

<Step title="Execute Transfer">
Submit the transaction with proper authentication and monitor for confirmation.
</Step>
</Steps>

## API Reference

### Endpoint
```http
POST /api/transactions/execute
```

### Headers

| Header | Format | Description | Example |
|--------|--------|-------------|---------|
| `Signing-Method` | `id:value` | Authentication using wallet owner's signing method | `756ae7a7-3713-43ee-9936-0dff50306488:123456` |

### Parameters

| Parameter | Type | Description | Required | Example |
|-----------|------|-------------|----------|---------|
| `transactionRequest` | Body | Container for all transaction details | ✅ | See examples below |
| `transactionRequest.type` | Body | Must be `"NFT_TRANSFER"` | ✅ | `"NFT_TRANSFER"` |
| `transactionRequest.walletId` | Body | ID of the wallet sending the NFT | ✅ | `"adc4c08a-b8fa-4e4c-z5a2-92c87b80f174"` |
| `transactionRequest.to` | Body | Recipient wallet address or email | ✅ | `"0xdc71b72db51e227e65a45004ab2798d31e8934c9"` |
| `transactionRequest.secretType` | Body | Blockchain network | ✅ | `"MATIC"` |
| `transactionRequest.tokenAddress` | Body | Contract address of the NFT | ✅ | `"0x4df47b4969b2911c966506e3592c41389493953b"` |
| `transactionRequest.tokenId` | Body | ID of the specific NFT within the contract | ✅ | `1` |

<Warning>
The deprecated `pincode` parameter should not be used. Always use the `Signing-Method` header for authentication.
</Warning>

## NFT Standards Support

<Accordion title="Supported NFT Standards">

**ERC-721 (Ethereum)**
- Each token is completely unique
- One owner per token
- Most common standard for digital collectibles
- Examples: CryptoPunks, Bored Ape Yacht Club

**ERC-1155 (Multi-Token)**
- Can represent both fungible and non-fungible tokens
- Batch operations supported
- More gas-efficient for multiple transfers
- Examples: Gaming items, trading cards

**Cross-Chain Support:**
- Ethereum mainnet
- Polygon (MATIC)
- Binance Smart Chain (BSC)
- Arbitrum
- Avalanche (AVAX)
- Optimism

**Transfer Behavior:**
- ERC-721: Transfers ownership of unique token
- ERC-1155: Transfers one copy of the specified token ID

</Accordion>

## Best Practices

<Warning>
**NFT Transfer Guidelines:**

1. **Verify Ownership**: Always confirm the wallet owns the NFT before transfer
2. **Validate Recipients**: Double-check recipient addresses to prevent loss
3. **Check Approvals**: Ensure proper approvals are set for marketplace transfers
4. **Monitor Gas Fees**: NFT transfers can be gas-intensive, especially on Ethereum
5. **Handle Errors**: Implement robust error handling for failed transfers
6. **Track Transactions**: Monitor transaction status after submission
7. **Test Transfers**: Use testnets for initial testing and validation
</Warning>

## Integration Examples

### NFT Marketplace Integration

```javascript
async function transferNFT(fromWalletId, toAddress, contractAddress, tokenId, blockchain) {
  // First verify ownership
  const isOwner = await verifyNFTOwnership(fromWalletId, contractAddress, tokenId);
  
  if (!isOwner) {
    throw new Error('Wallet does not own this NFT');
  }
  
  const transferData = {
    transactionRequest: {
      type: 'NFT_TRANSFER',
      walletId: fromWalletId,
      to: toAddress,
      secretType: blockchain,
      tokenAddress: contractAddress,
      tokenId: parseInt(tokenId)
    }
  };
  
  const response = await fetch('/api/transactions/execute', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Signing-Method': `${signingMethodId}:${signingMethodValue}`
    },
    body: JSON.stringify(transferData)
  });
  
  const result = await response.json();
  
  if (result.success) {
    console.log(`NFT transfer initiated: ${result.result.transactionHash}`);
    return result.result;
  } else {
    throw new Error(`Transfer failed: ${result.message}`);
  }
}
```

### Gaming Asset Transfer

```javascript
async function transferGameAsset(playerWalletId, recipientAddress, assetTokenId) {
  try {
    // Check if asset is transferable (not locked in game)
    const isTransferable = await checkAssetTransferability(assetTokenId);
    
    if (!isTransferable) {
      throw new Error('Asset is currently locked and cannot be transferred');
    }
    
    const transferResult = await transferNFT(
      playerWalletId,
      recipientAddress,
      process.env.GAME_ASSETS_CONTRACT,
      assetTokenId,
      'MATIC'
    );
    
    // Update game database
    await updateAssetOwnership(assetTokenId, recipientAddress);
    
    return transferResult;
  } catch (error) {
    console.error('Game asset transfer failed:', error);
    throw error;
  }
}
```

### Batch NFT Transfer

```javascript
async function batchTransferNFTs(transfers) {
  const results = [];
  
  for (const transfer of transfers) {
    try {
      const result = await transferNFT(
        transfer.walletId,
        transfer.recipient,
        transfer.contractAddress,
        transfer.tokenId,
        transfer.blockchain
      );
      
      results.push({
        tokenId: transfer.tokenId,
        recipient: transfer.recipient,
        success: true,
        transactionHash: result.transactionHash
      });
    } catch (error) {
      results.push({
        tokenId: transfer.tokenId,
        recipient: transfer.recipient,
        success: false,
        error: error.message
      });
    }
    
    // Add delay between transfers to avoid rate limiting
    await new Promise(resolve => setTimeout(resolve, 2000));
  }
  
  return results;
}
```

3. **Gas Estimation**: Check current gas prices before transfers

5. **Approval Management**: Set approvals efficiently for marketplace operations
