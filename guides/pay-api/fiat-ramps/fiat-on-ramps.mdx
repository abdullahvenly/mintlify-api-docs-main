## What are Fiat On-Ramps?

Fiat on-ramps allow users to purchase cryptocurrency using traditional fiat currencies (USD, EUR, GBP, etc.). Venly integrates with three leading providers to offer flexible options for your users.

## Transak On-Ramp

### API Endpoint

<CodeGroup>
```javascript JavaScript
const response = await fetch('/fiat-ramp/transak/on', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    fiatAmount: "300",
    fiatCurrency: "USD",
    cryptoAmount: "0.15",
    cryptoCurrency: {
      network: "ETHEREUM",
      symbol: "ETH"
    },
    redirectUrl: "https://www.venly.io/",
    email: "user@example.com",
    walletAddress: "0x3FE9a7D68C6a4a2d6575A6C803E4CDF9340E370E"
  })
});

const data = await response.json();
console.log('Transak URL:', data.result.url);
```

```python Python
import requests

response = requests.post(
    '/fiat-ramp/transak/on',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    json={
        'fiatAmount': '300',
        'fiatCurrency': 'USD',
        'cryptoAmount': '0.15',
        'cryptoCurrency': {
            'network': 'ETHEREUM',
            'symbol': 'ETH'
        },
        'redirectUrl': 'https://www.venly.io/',
        'email': 'user@example.com',
        'walletAddress': '0x3FE9a7D68C6a4a2d6575A6C803E4CDF9340E370E'
    }
)

data = response.json()
print('Transak URL:', data['result']['url'])
```

```bash cURL
curl -X POST '/fiat-ramp/transak/on' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "fiatAmount": "300",
    "fiatCurrency": "USD",
    "cryptoAmount": "0.15",
    "cryptoCurrency": {
      "network": "ETHEREUM",
      "symbol": "ETH"
    },
    "redirectUrl": "https://www.venly.io/",
    "email": "user@example.com",
    "walletAddress": "0x3FE9a7D68C6a4a2d6575A6C803E4CDF9340E370E"
  }'
```
</CodeGroup>

### Transak Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `fiatAmount` | number | ❌ | Amount in fiat currency user wants to spend |
| `fiatCurrency` | string | ❌ | Fiat currency code (USD, EUR, GBP, etc.) |
| `cryptoAmount` | number | ❌ | Amount of crypto user wants to buy |
| `cryptoCurrency.network` | string | ❌ | Blockchain network (ETHEREUM, BITCOIN, etc.) |
| `cryptoCurrency.symbol` | string | ❌ | Cryptocurrency symbol (ETH, BTC, etc.) |
| `redirectUrl` | string | ❌ | URL to redirect after purchase completion |
| `email` | string | ❌ | User's email address |
| `walletAddress` | string | ❌ | Destination wallet address |
| `isFeeCalculationHidden` | boolean | ❌ | Hide detailed fee breakdown |
| `hideMenu` | boolean | ❌ | Hide menu options completely |

<Tip>
**Transak Features:**
- If `fiatAmount` is passed, users can't change it later
- `cryptoAmount` is skipped if valid `fiatCurrency` and `fiatAmount` are provided
- Pre-filling email and wallet address improves user experience
</Tip>

### Moonpay Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `fiatAmount` | number | ❌ | Amount user wants to spend in fiat |
| `fiatCurrency` | string | ❌ | Fiat currency code (USD, EUR, GBP, etc.) |
| `cryptoAmount` | number | ❌ | Amount of crypto user wants to buy |
| `cryptoCurrency.symbol` | string | ❌ | Cryptocurrency symbol (required if walletAddress is passed) |
| `redirectUrl` | string | ❌ | Redirect URL after purchase (includes transaction ID and status) |
| `email` | string | ❌ | User's email address |
| `walletAddress` | string | ❌ | Destination wallet address |

<Note>
**Moonpay Priority:** If both `fiatAmount` and `cryptoAmount` are provided, `cryptoAmount` takes precedence.
</Note>

## Best Practices

<Tip>
**On-Ramp Optimization:**

1. **Provider Selection** - Choose based on user geography and preferences
2. **Pre-fill Information** - Use known user data to reduce friction
3. **Clear Communication** - Explain the process and expected timeframes
4. **Fallback Options** - Offer multiple providers for better success rates
5. **Mobile Optimization** - Ensure URLs work well on mobile devices
6. **Compliance** - Understand KYC/AML requirements for each provider
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Invalid currency combination">
    **Common issues:**
    - Unsupported fiat/crypto pair
    - Wrong network for cryptocurrency
    - Currency not available in user's region
    
    **Solution:** Use the [supported currencies endpoint](https://docs.venly.io/docs/on-ramp-off-ramp-supported-fiat-crypto-currencies) to verify valid combinations.
  </Accordion>
  
  <Accordion title="User verification failed">
    **Possible causes:**
    - Incomplete KYC documentation
    - Restricted jurisdiction
    - Payment method not supported
    
    **Solution:** Provide clear guidance on verification requirements and offer alternative providers.
  </Accordion>
  
  <Accordion title="Transaction timeout">
    **Common scenarios:**
    - User abandoned the flow
    - Payment processing delays
    - Network congestion
    
    **Solution:** Implement proper timeout handling and provide status updates to users.
  </Accordion>
  
  <Accordion title="Redirect URL issues">
    **Problems:**
    - URL not properly encoded
    - HTTPS requirement not met
    - Callback parameters not handled
    
    **Solution:** Ensure URLs are properly formatted and implement robust callback handling.
  </Accordion>
</AccordionGroup>
