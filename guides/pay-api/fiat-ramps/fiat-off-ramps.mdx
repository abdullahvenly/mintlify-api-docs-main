## What are Fiat Off-Ramps?

Fiat off-ramps allow users to sell cryptocurrency and receive traditional fiat currencies (USD, EUR, GBP, etc.) in return. Venly integrates with three leading providers to offer flexible options for your users to cash out their crypto holdings.

## Transak Off-Ramp

### API Endpoint

<CodeGroup>
```javascript JavaScript
const response = await fetch('/fiat-ramp/transak/off', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    fiatCurrency: "USD",
    cryptoAmount: "0.15",
    cryptoCurrency: {
      network: "mainnet",
      symbol: "BTC"
    },
    redirectUrl: "https://www.venly.io/",
    email: "user@example.com"
  })
});

const data = await response.json();
console.log('Transak off-ramp URL:', data.result.url);
```

### Transak Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `fiatCurrency` | string | ❌ | Fiat currency code for payout (USD, EUR, GBP, etc.) |
| `cryptoAmount` | number | ❌ | Amount of crypto user wants to sell |
| `cryptoCurrency.network` | string | ❌ | Blockchain network (bitcoin, ethereum, etc.) |
| `cryptoCurrency.symbol` | string | ❌ | Cryptocurrency symbol (BTC, ETH, etc.) |
| `redirectUrl` | string | ❌ | URL to redirect after sale completion |
| `email` | string | ❌ | User's email address |
| `isFeeCalculationHidden` | boolean | ❌ | Hide detailed fee breakdown |
| `hideMenu` | boolean | ❌ | Hide menu options completely |

<Tip>
**Transak Features:**
- If `cryptoAmount` is passed, users can't change it later
- Pre-filling email improves user experience
- Network parameter determines which blockchain to use
</Tip>

    class ScheduledSeller {
      constructor(apiKey) {
        this.apiKey = apiKey;
        this.schedules = new Map();
      }
    
      createSellingSchedule(userId, config) {
        const schedule = {
          userId,
          cryptoAsset: config.cryptoAsset,
          totalAmount: config.totalAmount,
          frequency: config.frequency, // 'daily', 'weekly', 'monthly'
          duration: config.duration, // number of periods
          targetFiat: config.targetFiat,
          provider: config.provider,
          nextSale: this.calculateNextSale(config.frequency),
          amountPerSale: config.totalAmount / config.duration,
          salesCompleted: 0
        };
        
        this.schedules.set(userId, schedule);
        this.scheduleNextSale(schedule);
        
        return schedule;
      }
    
      async executeScheduledSale(schedule) {
        try {
          const offrampUrl = await this.createOfframp({
            fiatCurrency: schedule.targetFiat,
            cryptoAmount: schedule.amountPerSale,
            cryptoCurrency: {
              symbol: schedule.cryptoAsset.symbol,
              network: schedule.cryptoAsset.network
            },
            email: schedule.userEmail,
            redirectUrl: `https://app.example.com/scheduled-sale/${schedule.userId}`
          }, schedule.provider);
          
          // Notify user about scheduled sale
          await this.notifyUser(schedule.userId, {
            type: 'scheduled_sale_ready',
            amount: schedule.amountPerSale,
            asset: schedule.cryptoAsset.symbol,
            url: offrampUrl
          });
          
          schedule.salesCompleted++;
          schedule.nextSale = this.calculateNextSale(schedule.frequency);
          
          if (schedule.salesCompleted < schedule.duration) {
            this.scheduleNextSale(schedule);
          } else {
            this.completeSchedule(schedule);
          }
          
        } catch (error) {
          console.error('Scheduled sale failed:', error);
          await this.handleSaleError(schedule, error);
        }
      }
    
      calculateNextSale(frequency) {
        const now = new Date();
        switch (frequency) {
          case 'daily':
            return new Date(now.getTime() + 24 * 60 * 60 * 1000);
          case 'weekly':
            return new Date(now.getTime() + 7 * 24 * 60 * 60 * 1000);
          case 'monthly':
            const nextMonth = new Date(now);
            nextMonth.setMonth(nextMonth.getMonth() + 1);
            return nextMonth;
          default:
            throw new Error('Invalid frequency');
        }
      }
    }

    class TaxOptimizedSelling {
      constructor(apiKey) {
        this.apiKey = apiKey;
      }
    
      async optimizeSaleForTaxes(holdings, targetAmount, taxYear) {
        // Sort holdings by tax efficiency (FIFO, LIFO, specific identification)
        const optimizedSales = this.calculateOptimalSales(holdings, targetAmount, taxYear);
        
        const salesPlan = [];
        
        for (const sale of optimizedSales) {
          const offrampUrl = await this.createOfframp(sale);
          
          salesPlan.push({
            ...sale,
            url: offrampUrl,
            taxImplications: {
              capitalGain: sale.salePrice - sale.costBasis,
              holdingPeriod: sale.holdingPeriod,
              taxTreatment: sale.holdingPeriod > 365 ? 'long-term' : 'short-term'
            }
          });
        }
        
        return {
          salesPlan,
          totalTaxLiability: this.calculateTotalTax(salesPlan),
          recommendations: this.generateTaxRecommendations(salesPlan)
        };
      }
    
      calculateOptimalSales(holdings, targetAmount, taxYear) {
        // Implement tax-loss harvesting logic
        // Consider holding periods for capital gains treatment
        // Optimize for minimal tax impact
        
        const sortedHoldings = holdings.sort((a, b) => {
          // Prioritize tax-efficient sales
          const aTaxEfficiency = this.calculateTaxEfficiency(a, taxYear);
          const bTaxEfficiency = this.calculateTaxEfficiency(b, taxYear);
          return bTaxEfficiency - aTaxEfficiency;
        });
        
        const sales = [];
        let remainingTarget = targetAmount;
        
        for (const holding of sortedHoldings) {
          if (remainingTarget <= 0) break;
          
          const saleAmount = Math.min(holding.amount, remainingTarget);
          sales.push({
            ...holding,
            saleAmount,
            salePrice: saleAmount * holding.currentPrice,
            costBasis: saleAmount * holding.averageCostBasis
          });
          
          remainingTarget -= saleAmount;
        }
        
        return sales;
      }
    }

    - Wrong network for cryptocurrency
    - Currency not available in user's region
    
    **Solution:** Use the [supported currencies endpoint](https://docs.venly.io/docs/on-ramp-off-ramp-supported-fiat-crypto-currencies) to verify valid combinations.
  </Accordion>
  
  <Accordion title="User verification failed">
    **Possible causes:**
    - Incomplete KYC documentation
    - Restricted jurisdiction
    - Payment method not supported
    
    **Solution:** Provide clear guidance on verification requirements and offer alternative providers.
  </Accordion>
  
  <Accordion title="Transaction timeout">
    **Common scenarios:**
    - User abandoned the flow
    - Payment processing delays
    - Network congestion
    
    **Solution:** Implement proper timeout handling and provide status updates to users.
  </Accordion>
  
  <Accordion title="Refund issues">
    **Problems:**
    - Missing refund wallet address
    - Invalid wallet address format
    - Network mismatch
    
    **Solution:** Always provide valid refund addresses and ensure network compatibility.
  </Accordion>
</AccordionGroup>
