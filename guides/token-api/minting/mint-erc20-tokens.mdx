## What is Token Minting?

Token minting is the process of creating new ERC20 tokens and distributing them to specified wallet addresses or email recipients. This is essential for token distribution, rewards, payments, and building token economies.

### Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `chain` | string | ✅ | The blockchain of the token contract |
| `contractAddress` | string | ✅ | The deployed token contract address |
| `destinations` | array | ✅ | Array of recipient addresses and amounts |
| `destinations.address` | string | ✅ | Wallet address or email address to receive tokens |
| `destinations.amount` | string | ✅ | Number of tokens to mint and send |

<Tip>
**Minting Guidelines:**
- Use string format for amounts to handle large numbers
- You can mix wallet addresses and email addresses
- Maximum recipients per batch varies by blockchain
- Ensure contract has sufficient remaining supply
</Tip>

## Minting Strategies

<AccordionGroup>
  <Accordion title="Equal Distribution" icon="balance-scale">
    **Same amount to all recipients:**
    ```json
    {
      "destinations": [
        {"address": "0x123...", "amount": "100"},

        {"address": "0x789...", "amount": "100"}

    }
    ```

    Perfect for airdrops and equal rewards.
  </Accordion>
  
  <Accordion title="Tiered Distribution" icon="layer-group">
    **Different amounts based on tiers:**
    ```json
    {
      "destinations": [
        {"address": "0x123...", "amount": "1000"},  // VIP tier
        {"address": "0x456...", "amount": "500"},   // Premium tier
        {"address": "0x789...", "amount": "100"}    // Basic tier
      ]
    }
    ```
    Great for loyalty programs and membership levels.
  </Accordion>
  
  <Accordion title="Email Distribution" icon="envelope">
    **Send to email addresses:**
    ```json
    {
      "destinations": [
        {"address": "user1@example.com", "amount": "50"},
        {"address": "user2@example.com", "amount": "75"},
        {"address": "0x123...", "amount": "100"}
      ]
    }
    ```
    Mix wallet addresses and emails for broader reach.
  </Accordion>
  
  <Accordion title="Large Scale Distribution" icon="users">
    **Programmatic batch creation:**
    ```javascript
    const destinations = [];
    const recipients = [
      {address: "0x123...", amount: "100"},
      {address: "user@email.com", amount: "50"},
      // ... more recipients
    ];

      destinations.push({
        address: recipient.address,
        amount: recipient.amount.toString()
      });
    });
    ```
  </Accordion>

## Monitor Minting Status

Track individual mint operations:

<CodeGroup>
```javascript JavaScript
// Track each mint individually
const mintIds = data.result.mints.map(mint => mint.id);

for (const mintId of mintIds) {
  const statusResponse = await fetch(`/api/v3/erc20/tokens/mints/${mintId}`, {
    headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
  });
  
  const status = await statusResponse.json();
  console.log(`Mint ${mintId}: ${status.result.status}`);
}
```

```python Python
# Track each mint individually
mint_ids = [mint['id'] for mint in data['result']['mints']]

for mint_id in mint_ids:
    status_response = requests.get(
        f'/api/v3/erc20/tokens/mints/{mint_id}',
        headers={'Authorization': 'Bearer YOUR_API_KEY'}
    )
    
    status = status_response.json()
    print(f"Mint {mint_id}: {status['result']['status']}")
```
</CodeGroup>

## Best Practices

<Tip>
**Token Minting Optimization:**

1. **Supply Management** - Check remaining supply before large mints
2. **Batch Sizing** - Use optimal batch sizes for your blockchain
3. **Address Validation** - Verify all recipient addresses before minting
4. **Amount Formatting** - Use string format for large token amounts
5. **Status Monitoring** - Track mint progress for important distributions
6. **Error Handling** - Implement retry logic for failed mints
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Insufficient supply">
    **Error:** Not enough tokens remaining in contract supply
    
    **Causes:**
    - Trying to mint more than max supply allows
    - Contract supply already exhausted
    - Previous mints consumed available supply
    
    **Solution:** Check contract supply limits and adjust mint amounts accordingly.
  </Accordion>
  
  <Accordion title="Invalid recipient address">
    **Common issues:**
    - Malformed wallet address
    - Wrong blockchain address format
    - Invalid email address format
    
    **Solution:** Validate all addresses before submitting mint request.
  </Accordion>
  
  <Accordion title="Mint transaction failed">
    **Possible causes:**
    - Network congestion
    - Insufficient gas fees
    - Contract permissions issue
    - Blockchain connectivity problems
    
    **Solution:** Check mint status, verify contract permissions, and retry if necessary.
  </Accordion>
  
  <Accordion title="Some mints succeeded, others failed">
    **Batch processing behavior:**
    - Each destination is processed individually
    - Some can succeed while others fail
    - Check individual mint statuses
    
    **Solution:** Review failed mint IDs and retry with corrected parameters.
  </Accordion>
</AccordionGroup>

## Integration Examples

<CodeGroup>
```javascript JavaScript
// Complete minting workflow
async function distributeTokens(contractAddress, recipients) {
  try {
    // Prepare destinations
    const destinations = recipients.map(recipient => ({
      address: recipient.address,
      amount: recipient.amount.toString()
    }));
    
    // Mint tokens
    const response = await fetch('/api/v3/erc20/tokens/mints', {
      method: 'POST',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        chain: "MATIC",
        contractAddress,
        destinations
      })
    });
    
    const data = await response.json();
    
    // Track minting progress
    const mintIds = data.result.mints.map(mint => mint.id);
    console.log(`Started ${mintIds.length} mint operations`);
    
    return mintIds;
  } catch (error) {
    console.error('Minting failed:', error);
    throw error;
  }
}

// Usage
const recipients = [
  { address: "0x123...", amount: 100 },
  { address: "user@email.com", amount: 50 }
];

const mintIds = await distributeTokens("0xcontract...", recipients);
```

```python Python
import requests
import time

def distribute_tokens(contract_address, recipients):
    """Distribute tokens to multiple recipients"""
    try:
        # Prepare destinations
        destinations = [
            {
                'address': recipient['address'],
                'amount': str(recipient['amount'])
            }
            for recipient in recipients
        ]
        
        # Mint tokens
        response = requests.post(
            '/api/v3/erc20/tokens/mints',
            headers={
                'Authorization': 'Bearer YOUR_API_KEY',
                'Content-Type': 'application/json'
            },
            json={
                'chain': 'MATIC',
                'contractAddress': contract_address,
                'destinations': destinations
            }
        )
        
        data = response.json()
        
        # Track minting progress
        mint_ids = [mint['id'] for mint in data['result']['mints']]
        print(f'Started {len(mint_ids)} mint operations')
        
        return mint_ids
    except Exception as error:
        print(f'Minting failed: {error}')
        raise error

# Usage
recipients = [
    {'address': '0x123...', 'amount': 100},
    {'address': 'user@email.com', 'amount': 50}
]

mint_ids = distribute_tokens('0xcontract...', recipients)
```
