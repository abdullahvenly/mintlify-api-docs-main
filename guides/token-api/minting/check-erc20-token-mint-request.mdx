## What is Mint Status Checking?

After initiating ERC20 token minting operations, each mint request is processed asynchronously on the blockchain. This endpoint allows you to track the progress of individual mint operations and verify successful token distribution.

<Steps>

</Note>

## Check Mint Status

### API Endpoint

<CodeGroup>
```javascript JavaScript
const mintId = "7f78e6ed-d733-430c-83a4-5fc28032e667";

const response = await fetch(`/api/v3/erc20/tokens/mints/${mintId}`, {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const data = await response.json();
console.log('Mint status:', data.result.status);
console.log('Transaction hash:', data.result.transactionHash);
```

## Batch Status Monitoring

Monitor multiple mint operations simultaneously:

<CodeGroup>
```javascript JavaScript
async function checkMultipleMints(mintIds) {
  const promises = mintIds.map(async (mintId) => {
    try {
      const response = await fetch(`/api/v3/erc20/tokens/mints/${mintId}`, {
        headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
      });
      
      const data = await response.json();
      return {
        mintId,
        status: data.result.status,
        destination: data.result.destination,
        transactionHash: data.result.transactionHash
      };
    } catch (error) {
      return {
        mintId,
        error: error.message
      };
    }
  });
  
  const results = await Promise.all(promises);
  
  const succeeded = results.filter(r => r.status === 'SUCCEEDED');
  const pending = results.filter(r => r.status === 'PENDING');
  const failed = results.filter(r => r.status === 'FAILED' || r.error);
  
  console.log(`✅ Succeeded: ${succeeded.length}`);
  console.log(`⏳ Pending: ${pending.length}`);
  console.log(`❌ Failed: ${failed.length}`);
  
  return { succeeded, pending, failed };
}

// Usage
const mintIds = [
  "7f78e6ed-d733-430c-83a4-5fc28032e667",
  "d06ff454-b603-4279-8439-39d6949abf67"
];

const results = await checkMultipleMints(mintIds);
```

```python Python
import requests
import asyncio
import aiohttp

async def check_multiple_mints(mint_ids):
    async with aiohttp.ClientSession() as session:
        tasks = []
        
        for mint_id in mint_ids:
            task = check_single_mint(session, mint_id)
            tasks.append(task)
        
        results = await asyncio.gather(*tasks, return_exceptions=True)
        
        succeeded = [r for r in results if r.get('status') == 'SUCCEEDED']
        pending = [r for r in results if r.get('status') == 'PENDING']
        failed = [r for r in results if r.get('status') == 'FAILED' or isinstance(r, Exception)]
        
        print(f"✅ Succeeded: {len(succeeded)}")
        print(f"⏳ Pending: {len(pending)}")
        print(f"❌ Failed: {len(failed)}")
        
        return {'succeeded': succeeded, 'pending': pending, 'failed': failed}

async def check_single_mint(session, mint_id):
    try:
        async with session.get(
            f'/api/v3/erc20/tokens/mints/{mint_id}',
            headers={'Authorization': 'Bearer YOUR_API_KEY'}
        ) as response:
            data = await response.json()
            return {
                'mintId': mint_id,
                'status': data['result']['status'],
                'destination': data['result']['destination'],
                'transactionHash': data['result'].get('transactionHash')
            }
    except Exception as error:
        return {'mintId': mint_id, 'error': str(error)}

# Usage
mint_ids = [
    "7f78e6ed-d733-430c-83a4-5fc28032e667",
    "d06ff454-b603-4279-8439-39d6949abf67"
]

results = asyncio.run(check_multiple_mints(mint_ids))
```

## Polling for Status Updates

Implement polling to wait for mint completion:

<CodeGroup>
```javascript JavaScript
async function waitForMintCompletion(mintId, maxAttempts = 20) {
  for (let attempt = 1; attempt <= maxAttempts; attempt++) {
    try {
      const response = await fetch(`/api/v3/erc20/tokens/mints/${mintId}`, {
        headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
      });
      
      const data = await response.json();
      const status = data.result.status;
      
      console.log(`Attempt ${attempt}: Mint ${mintId} status is ${status}`);
      
      if (status === 'SUCCEEDED') {
        console.log('✅ Mint completed successfully!');
        console.log('Transaction:', data.result.transactionHash);
        console.log('Recipient:', data.result.destination.address);
        console.log('Amount:', data.result.destination.amount);
        return data.result;
      }
      
      if (status === 'FAILED') {
        throw new Error('Mint operation failed');
      }
      
      // Wait 5 seconds before next check
      await new Promise(resolve => setTimeout(resolve, 5000));
      
    } catch (error) {
      console.error(`Error checking mint status: ${error.message}`);
      throw error;
    }
  }
  
  throw new Error('Mint timeout - max attempts reached');
}

// Usage
const mintId = "7f78e6ed-d733-430c-83a4-5fc28032e667";
const result = await waitForMintCompletion(mintId);
```

```python Python
import requests
import time

def wait_for_mint_completion(mint_id, max_attempts=20):
    for attempt in range(1, max_attempts + 1):
        try:
            response = requests.get(
                f'/api/v3/erc20/tokens/mints/{mint_id}',
                headers={'Authorization': 'Bearer YOUR_API_KEY'}
            )
            
            data = response.json()
            status = data['result']['status']
            
            print(f'Attempt {attempt}: Mint {mint_id} status is {status}')
            
            if status == 'SUCCEEDED':
                print('✅ Mint completed successfully!')
                print('Transaction:', data['result']['transactionHash'])
                print('Recipient:', data['result']['destination']['address'])
                print('Amount:', data['result']['destination']['amount'])
                return data['result']
            
            if status == 'FAILED':
                raise Exception('Mint operation failed')
            
            # Wait 5 seconds before next check
            time.sleep(5)
            
        except Exception as error:
            print(f'Error checking mint status: {error}')
            raise error
    
    raise Exception('Mint timeout - max attempts reached')

# Usage
mint_id = "7f78e6ed-d733-430c-83a4-5fc28032e667"
result = wait_for_mint_completion(mint_id)
```

    class MintMonitor {
      constructor(apiKey) {
        this.apiKey = apiKey;
        this.activeMints = new Map();
      }
    
      async addMint(mintId, metadata = {}) {
        this.activeMints.set(mintId, {
          ...metadata,
          status: 'PENDING',
          attempts: 0,
          startTime: Date.now()
        });
        
        this.checkMint(mintId);
      }
    
      async checkMint(mintId) {
        const mint = this.activeMints.get(mintId);
        if (!mint || mint.attempts >= 20) return;
    
        try {
          const response = await fetch(`/api/v3/erc20/tokens/mints/${mintId}`, {
            headers: { 'Authorization': `Bearer ${this.apiKey}` }
          });
    
          const data = await response.json();
          const status = data.result.status;
    
          mint.status = status;
          mint.attempts++;
          mint.lastCheck = Date.now();
    
          if (status === 'SUCCEEDED' || status === 'FAILED') {
            mint.completedAt = Date.now();
            mint.duration = mint.completedAt - mint.startTime;
            mint.transactionHash = data.result.transactionHash;
            
            this.onMintComplete(mintId, data.result);
          } else {
            // Continue polling
            setTimeout(() => this.checkMint(mintId), 5000);
          }
        } catch (error) {
          mint.error = error.message;
        }
      }
    
      onMintComplete(mintId, result) {
        console.log(`Mint ${mintId} completed:`, result.status);
        // Emit event, update UI, send notification, etc.
      }
    
      getStatus() {
        const mints = Array.from(this.activeMints.entries());
        return {
          total: mints.length,
          pending: mints.filter(([_, m]) => m.status === 'PENDING').length,
          succeeded: mints.filter(([_, m]) => m.status === 'SUCCEEDED').length,
          failed: mints.filter(([_, m]) => m.status === 'FAILED').length
        };
      }
    }
    
    // Usage
    const monitor = new MintMonitor('YOUR_API_KEY');
    monitor.addMint('mint-id-1', { recipient: 'user@email.com', amount: 100 });
    monitor.addMint('mint-id-2', { recipient: '0x123...', amount: 50 });

## Troubleshooting

<AccordionGroup>
  <Accordion title="Mint ID not found">
    **Error:** Invalid or non-existent mint ID
    
    **Causes:**
    - Incorrect mint ID format
    - Mint ID from different environment
    - Typo in mint ID
    
    **Solution:** Verify the mint ID from your token minting response and ensure you're using the correct environment.
  </Accordion>
  
  <Accordion title="Status stuck on PENDING">
    **Common causes:**
    - Network congestion
    - Insufficient gas fees
    - Blockchain connectivity issues
    - Large batch processing delays
    
    **Solution:** Wait longer or check the blockchain explorer. Some networks may take several minutes during high congestion.
  </Accordion>
  
  <Accordion title="Mint failed">
    **Possible reasons:**
    - Insufficient contract supply
    - Invalid recipient address
    - Network issues during processing
    - Contract permissions problem
    
    **Solution:** Check the contract supply, verify recipient addresses, and consider retrying the mint operation.
  </Accordion>
  
  <Accordion title="Missing transaction hash">
    **Issue:** Status is SUCCEEDED but no transaction hash
    
    **Cause:** Response timing or API version issue
    
    **Solution:** The transaction hash may appear shortly after status changes to SUCCEEDED. Check again or use blockchain explorer with contract address.
  </Accordion>
</AccordionGroup>

## Best Practices

<Tip>
**Mint Status Monitoring Best Practices:**

1. **Track All Mint IDs** - Save all mint IDs from batch operations
2. **Implement Reasonable Timeouts** - Set appropriate polling limits
3. **Handle Partial Failures** - Some mints may succeed while others fail
4. **Use Webhooks When Possible** - Reduce API calls with real-time notifications
5. **Monitor Transaction Hashes** - Verify on blockchain explorers for additional confirmation
6. **Implement Retry Logic** - Handle failed mints gracefully with retry mechanisms
</Tip>
