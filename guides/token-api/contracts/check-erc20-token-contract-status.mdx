## What is Contract Status Checking?

After creating an ERC20 token contract, the deployment process is asynchronous. This endpoint allows you to monitor the deployment progress and retrieve the final contract address once deployment is complete.

## Check Contract Status

### API Endpoint

<CodeGroup>
```javascript JavaScript
const deploymentId = "7eae57cb-6769-4a2a-84c8-32e72f0aaee7";

const response = await fetch(`/api/v3/erc20/contracts/deployments/${deploymentId}`, {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const data = await response.json();
console.log('Contract status:', data.result.status);
console.log('Contract address:', data.result.address);
```

```python Python
import requests

deployment_id = "7eae57cb-6769-4a2a-84c8-32e72f0aaee7"

response = requests.get(
    f'/api/v3/erc20/contracts/deployments/{deployment_id}',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)

data = response.json()
print('Contract status:', data['result']['status'])
print('Contract address:', data['result']['address'])
```

## Polling for Status Updates

Implement polling to automatically check status until deployment completes:

<CodeGroup>
```javascript JavaScript
async function waitForContractDeployment(deploymentId, maxAttempts = 30) {
  for (let attempt = 1; attempt <= maxAttempts; attempt++) {
    try {
      const response = await fetch(`/api/v3/erc20/contracts/deployments/${deploymentId}`, {
        headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
      });
      
      const data = await response.json();
      const status = data.result.status;
      
      console.log(`Attempt ${attempt}: Status is ${status}`);
      
      if (status === 'SUCCEEDED') {
        console.log('Contract deployed successfully!');
        console.log('Contract address:', data.result.address);
        return data.result;
      }
      
      if (status === 'FAILED') {
        throw new Error('Contract deployment failed');
      }
      
      // Wait 10 seconds before next check
      await new Promise(resolve => setTimeout(resolve, 10000));
      
    } catch (error) {
      console.error(`Error checking status: ${error.message}`);
      throw error;
    }
  }
  
  throw new Error('Deployment timeout - max attempts reached');
}

// Usage
const deploymentId = "7eae57cb-6769-4a2a-84c8-32e72f0aaee7";
const contract = await waitForContractDeployment(deploymentId);
```

```python Python
import requests
import time

def wait_for_contract_deployment(deployment_id, max_attempts=30):
    for attempt in range(1, max_attempts + 1):
        try:
            response = requests.get(
                f'/api/v3/erc20/contracts/deployments/{deployment_id}',
                headers={'Authorization': 'Bearer YOUR_API_KEY'}
            )
            
            data = response.json()
            status = data['result']['status']
            
            print(f'Attempt {attempt}: Status is {status}')
            
            if status == 'SUCCEEDED':
                print('Contract deployed successfully!')
                print('Contract address:', data['result']['address'])
                return data['result']
            
            if status == 'FAILED':
                raise Exception('Contract deployment failed')
            
            # Wait 10 seconds before next check
            time.sleep(10)
            
        except Exception as error:
            print(f'Error checking status: {error}')
            raise error
    
    raise Exception('Deployment timeout - max attempts reached')

# Usage
deployment_id = "7eae57cb-6769-4a2a-84c8-32e72f0aaee7"
contract = wait_for_contract_deployment(deployment_id)
```

## Status Monitoring Strategies

<AccordionGroup>
  <Accordion title="Polling Intervals" icon="clock">
    **Recommended polling strategies:**
    
    ```javascript
    // Conservative approach (recommended)
    const pollInterval = 10000; // 10 seconds
    
    // Aggressive approach (for urgent deployments)
    const pollInterval = 5000; // 5 seconds
    
    // Exponential backoff
    let pollInterval = 5000;
    const maxInterval = 30000;
    
    function increasePollInterval() {
      pollInterval = Math.min(pollInterval * 1.5, maxInterval);
    }
    ```
    
    **Guidelines:**
    - Start with 10-second intervals
    - Increase interval if deployment takes longer
    - Set maximum attempts to avoid infinite loops
  </Accordion>
  
  <Accordion title="Error Handling" icon="exclamation-triangle">
    **Handle different failure scenarios:**
    
    ```javascript
    async function checkContractStatus(deploymentId) {
      try {
        const response = await fetch(`/api/v3/erc20/contracts/deployments/${deploymentId}`, {
          headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
        });
        
        if (!response.ok) {
          throw new Error(`HTTP ${response.status}: ${response.statusText}`);
        }
        
        const data = await response.json();
        
        if (!data.success) {
          throw new Error('API request failed');
        }
        
        return data.result;
        
      } catch (error) {
        console.error('Status check failed:', error.message);
        throw error;
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Webhook Alternative" icon="webhook">
    **Use webhooks for real-time updates:**
    
    Instead of polling, consider setting up [webhooks](https://docs.venly.io/docs/webhooks-token-api) to receive automatic notifications when contract deployment completes.

        "status": "SUCCEEDED"

  </Accordion>
  
  <Accordion title="Batch Status Checking" icon="list">
    **Check multiple contracts simultaneously:**
    
    ```javascript
    async function checkMultipleContracts(deploymentIds) {
      const promises = deploymentIds.map(id => 
        checkContractStatus(id).catch(error => ({ id, error }))
      );
      
      const results = await Promise.all(promises);
      
      const succeeded = results.filter(r => r.status === 'SUCCEEDED');
      const pending = results.filter(r => r.status === 'PENDING');
      const failed = results.filter(r => r.status === 'FAILED' || r.error);
      
      return { succeeded, pending, failed };
    }
    ```
  </Accordion>

  <Tab title="Node.js Service">
    ```javascript
    class ContractDeploymentTracker {
      constructor(apiKey) {
        this.apiKey = apiKey;
        this.deployments = new Map();
      }
    
      async trackDeployment(deploymentId, callback) {
        this.deployments.set(deploymentId, { callback, attempts: 0 });
        await this.checkDeployment(deploymentId);
      }
    
      async checkDeployment(deploymentId) {
        const deployment = this.deployments.get(deploymentId);
        if (!deployment) return;
    
        try {
          const response = await fetch(`/api/v3/erc20/contracts/deployments/${deploymentId}`, {
            headers: { 'Authorization': `Bearer ${this.apiKey}` }
          });
    
          const data = await response.json();
          const status = data.result.status;
    
          if (status === 'SUCCEEDED' || status === 'FAILED') {
            deployment.callback(data.result);
            this.deployments.delete(deploymentId);
          } else if (deployment.attempts < 30) {
            deployment.attempts++;
            setTimeout(() => this.checkDeployment(deploymentId), 10000);
          } else {
            deployment.callback({ error: 'Timeout' });
            this.deployments.delete(deploymentId);
          }
        } catch (error) {
          deployment.callback({ error: error.message });
          this.deployments.delete(deploymentId);
        }
      }
    }
    
    // Usage
    const tracker = new ContractDeploymentTracker('YOUR_API_KEY');
    tracker.trackDeployment('deployment-id', (result) => {
      if (result.error) {
        console.error('Deployment failed:', result.error);
      } else {
        console.log('Contract deployed:', result.address);
      }
    });
    ```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Deployment ID not found">
    **Error:** Invalid or non-existent deployment ID
    
    **Causes:**
    - Incorrect deployment ID format
    - Deployment ID from different environment
    - Typo in deployment ID
    
    **Solution:** Verify the deployment ID from your contract creation response and ensure you're using the correct environment.
  </Accordion>
  
  <Accordion title="Status stuck on PENDING">
    **Common causes:**
    - Network congestion
    - Insufficient gas fees
    - Blockchain connectivity issues
    
    **Solution:** Wait longer or check blockchain explorer using the transaction hash. Some networks may take several minutes during high congestion.
  </Accordion>
  
  <Accordion title="Deployment failed">
    **Possible reasons:**
    - Invalid contract parameters
    - Network issues during deployment
    - Insufficient funds for gas fees
    
    **Solution:** Check the transaction hash on a blockchain explorer for detailed error information, then create a new contract with corrected parameters.
  </Accordion>
  
  <Accordion title="Missing contract address">
    **Issue:** Status is SUCCEEDED but no address field
    
    **Cause:** Response parsing error or API version mismatch
    
    **Solution:** Ensure you're using the latest API version and check the complete response structure.
  </Accordion>
</AccordionGroup>

## Best Practices

<Tip>
**Status Monitoring Best Practices:**

1. **Save Deployment IDs** - Always store deployment IDs for future reference
2. **Implement Timeouts** - Set maximum polling attempts to avoid infinite loops
3. **Use Exponential Backoff** - Increase polling intervals for long-running deployments
4. **Handle All States** - Account for PENDING, SUCCEEDED, and FAILED states
5. **Store Contract Addresses** - Save the final contract address for future operations
6. **Monitor Transaction Hash** - Use blockchain explorers for additional debugging
</Tip>
