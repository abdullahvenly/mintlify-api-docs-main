## 1. Authentication

You will need your access credentials (**Client ID** and **Client Secret**) to obtain a `bearer token` and authorize all API calls.

### Get Bearer Token

<CodeGroup>
```bash cURL
curl -X POST 'https://login-sandbox.venly.io/auth/realms/Venly/protocol/openid-connect/token' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'grant_type=client_credentials' \
  -d 'client_id=YOUR_CLIENT_ID' \
  -d 'client_secret=YOUR_CLIENT_SECRET'
```

```javascript JavaScript
const getAccessToken = async () => {
  const response = await fetch('https://login-sandbox.venly.io/auth/realms/Venly/protocol/openid-connect/token', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded',
    },
    body: new URLSearchParams({
      'grant_type': 'client_credentials',
      'client_id': 'YOUR_CLIENT_ID',
      'client_secret': 'YOUR_CLIENT_SECRET'
    })
  });
  
  const data = await response.json();
  return data.access_token;
};
```

```python Python
import requests

def get_access_token():
    url = 'https://login-sandbox.venly.io/auth/realms/Venly/protocol/openid-connect/token'
    
    data = {
        'grant_type': 'client_credentials',
        'client_id': 'YOUR_CLIENT_ID',
        'client_secret': 'YOUR_CLIENT_SECRET'
    }
    
    response = requests.post(url, data=data)
    return response.json()['access_token']
```
</CodeGroup>

<Tip>
Save the `access_token` from the response - you'll need it for all subsequent API calls as a Bearer token.
</Tip>

## 2. Creating a Contract

Let's create your first NFT contract (collection) on the **Polygon** (_MATIC_) testnet chain.

### API Endpoint
```http
POST /api/v3/erc1155/contracts/deployments
```

### Request Parameters

| Parameter | Description | Type | Required |
|-----------|-------------|------|----------|
| `name` | The name of your NFT collection/contract | String | ‚úÖ |
| `description` | The description of your NFT collection/contract | String | ‚úÖ |
| `image` | The image URL for your NFT collection/contract | String | ‚úÖ |
| `externalUrl` | Link to your website, landing page, etc. | String | ‚úÖ |
| `chain` | The blockchain to create the contract on | String | ‚úÖ |

### Example Request

<CodeGroup>
```bash cURL
curl -X POST 'https://nft-api-sandbox.venly.io/api/v3/erc1155/contracts/deployments' \
  -H 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "Venly NFT Collection",
    "description": "This collection contains Venly NFTs.",
    "image": "https://storage-staging.venly.io/applications/bff93371-db40-4363-9692-4fbe64a34b54/logovenly.png",
    "externalUrl": "https://www.venly.io/",
    "chain": "MATIC"
  }'
```

## 4. Creating Token-type (NFT Template)

Create a token-type which serves as a template for minting NFTs. This template defines the NFT's metadata, attributes, and properties.

### API Endpoint
```http
POST /api/v3/erc1155/token-types/creations
```

### Request Parameters

| Parameter | Description | Type | Required |
|-----------|-------------|------|----------|
| `chain` | The blockchain of the contract | String | ‚úÖ |
| `contractAddress` | The contract address to create the token-type under | String | ‚úÖ |
| `creations` | Array of token-type details | Array | ‚úÖ |
| `creations.name` | The name of the token-type | String | ‚úÖ |
| `creations.description` | The description of the token-type | String | ‚ùå |
| `creations.image` | The image URL for the token-type | String | ‚ùå |

## 6. Minting an NFT

Now let's mint your first NFT using the token-type template you created!

### API Endpoint
```http
POST /api/v3/erc1155/tokens/mints
```

### Request Parameters

| Parameter | Description | Type | Required |
|-----------|-------------|------|----------|
| `contractAddress` | The contract address | String | ‚úÖ |
| `chain` | The blockchain of the contract | String | ‚úÖ |
| `tokenTypeId` | The ID of the token-type template | String | ‚úÖ |
| `destinations` | Array of wallet addresses and amounts | Array | ‚úÖ |
| `destinations.address` | Wallet address to mint and send the NFT | String | ‚úÖ |
| `destinations.amount` | Number of NFTs to mint and send | Integer | ‚úÖ |

### Example Request

<CodeGroup>
```bash cURL
curl -X POST 'https://nft-api-sandbox.venly.io/api/v3/erc1155/tokens/mints' \
  -H 'Authorization: Bearer YOUR_ACCESS_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "contractAddress": "0xf0b30ec3720e9be1452b64d2ee476f57c2f535d3",
    "chain": "MATIC",
    "tokenTypeId": "2",
    "destinations": [
        {
            "address": "0x9A9F6A560bBdCFbbb014303fc817D1e5EeaA3AE2",
            "amount": 1
        }
    ]
  }'
```
