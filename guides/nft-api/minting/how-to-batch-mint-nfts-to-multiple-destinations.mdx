<CodeGroup>
```javascript JavaScript

  method: 'POST',
  headers: {

  },
  body: JSON.stringify({
    contractAddress: "0x30d6cff9cb268c59c75a94755b2c60e118d65657",
    chain: "MATIC",
    tokenTypeId: "3",

      {
```

### Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `contractAddress` | string | ✅ | The contract address |
| `chain` | string | ✅ | The blockchain of the contract |
| `tokenTypeId` | string | ✅ | The ID of the token-type (from token-type creation response) |
| `destinations` | array | ✅ | Array of wallet/email addresses and amounts to mint |
| `destinations.address` | string | ✅ | The wallet address to mint and send the NFT |
| `destinations.amount` | integer | ✅ | The number of NFTs you want to mint and send |

<Tip>
**Scaling Up:** You can add up to **100 different wallet addresses** by following the same syntax in the destinations array. Each destination can specify a different amount of NFTs to mint.
</Tip>

## Monitoring Batch Mints

### Track Individual Mint Status

<CodeGroup>
```javascript JavaScript
// Track each mint individually
const mintIds = data.result.mints.map(mint => mint.id);

for (const mintId of mintIds) {
  const statusResponse = await fetch(`/api/v3/erc1155/tokens/mints/${mintId}`, {
    headers: { 'Authorization': 'Bearer YOUR_API_KEY' }
  });
  
  const status = await statusResponse.json();
  console.log(`Mint ${mintId}: ${status.result.status}`);
}
```

```python Python
# Track each mint individually
mint_ids = [mint['id'] for mint in data['result']['mints']]

for mint_id in mint_ids:
    status_response = requests.get(
        f'/api/v3/erc1155/tokens/mints/{mint_id}',
        headers={'Authorization': 'Bearer YOUR_API_KEY'}
    )
    
    status = status_response.json()
    print(f"Mint {mint_id}: {status['result']['status']}")
```
</CodeGroup>

### Batch Status Summary

<Tabs>
  <Tab title="Status Types">
    | Status | Description |
    |--------|-------------|
    | `PENDING` | Mint is being processed on the blockchain |
    | `SUCCESS` | NFT successfully minted and transferred |
    | `FAILED` | Mint failed - check error details |

## Best Practices for Batch Minting

<Tip>
**Optimization Strategies:**

1. **Batch Size Management** - Use optimal batch sizes (50-100 recipients)
2. **Gas Fee Optimization** - Mint during low network activity
3. **Error Handling** - Implement retry logic for failed mints
4. **Status Monitoring** - Track individual mint progress
5. **Address Validation** - Verify all recipient addresses before minting
6. **Amount Planning** - Calculate total supply requirements
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Some mints failed">
    **Possible causes:**
    - Invalid recipient addresses
    - Insufficient token supply
    - Network congestion
    - Gas estimation errors
    
    **Solution:** Check individual mint statuses and retry failed mints with corrected parameters.
  </Accordion>
  
  <Accordion title="High gas fees">
    **Optimization strategies:**
    - Use Layer 2 solutions like Polygon
    - Mint during off-peak hours
    - Optimize batch sizes
    - Consider fungible tokens for identical items
    
    **Solution:** Monitor network conditions and adjust timing accordingly.
  </Accordion>
  
  <Accordion title="Address validation errors">
    **Common issues:**
    - Invalid address format
    - Mixing different blockchain addresses
    - Using contract addresses instead of wallet addresses
    
    **Solution:** Validate all addresses before submitting the batch mint request.
  </Accordion>
  
  <Accordion title="Batch size limitations">
    **Current limits:**
    - Maximum 100 destinations per batch
    - Each destination can specify different amounts
    - Total gas limit considerations
    
    **Solution:** Split large distributions into multiple batches if needed.
  </Accordion>
</AccordionGroup>
