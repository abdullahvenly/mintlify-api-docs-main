## What is a Token-Type?

A token-type is essentially an **NFT template** that defines the characteristics of your NFTs before they're minted. Think of it as creating a blueprint that you can use to mint multiple NFTs with the same properties.

<CardGroup cols={2}>
  <Card title="Template Definition" icon="file-text">
    Define name, image, description, and attributes once
  </Card>
  <Card title="Multiple Minting" icon="copy">
    Use the same template to mint many NFTs
  </Card>
  <Card title="Consistent Metadata" icon="database">
    All NFTs from the template share the same base properties
  </Card>
  <Card title="Efficient Process" icon="zap">
    Create once, mint many times
  </Card>
</CardGroup>

<Note>
This means you just have to define the NFT template once with parameters like its `name`, `image`, and `description`. Using this token-type you can mint multiple NFTs with identical characteristics.
</Note>

## Create Your Token-Type

### API Endpoint

<CodeGroup>
```javascript JavaScript
const response = await fetch('/api/v3/erc1155/token-types/creations', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    chain: "MATIC",
    contractAddress: "0xf5b11b4f458cc12a7989a146c5db2e7d500e2241",
    creations: [
      {
        name: "My first NFT Token-type",
        description: "Venly",
        image: "https://static.wikia.nocookie.net/parody/images/4/42/74915084_10162764640400387_6139958579186106368_o.jpg"
      }
    ]
  })
});

const data = await response.json();
console.log('Token-type created:', data);
```

```python Python
import requests

response = requests.post(
    '/api/v3/erc1155/token-types/creations',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    json={
        'chain': 'MATIC',
        'contractAddress': '0xf5b11b4f458cc12a7989a146c5db2e7d500e2241',
        'creations': [
            {
                'name': 'My first NFT Token-type',
                'description': 'Venly',
                'image': 'https://static.wikia.nocookie.net/parody/images/4/42/74915084_10162764640400387_6139958579186106368_o.jpg'
            }
        ]
    }
)

data = response.json()
print('Token-type created:', data)
```

```bash cURL

    "creations": [
      {
        "name": "My first NFT Token-type",
        "description": "Venly",
        "image": "https://static.wikia.nocookie.net/parody/images/4/42/74915084_10162764640400387_6139958579186106368_o.jpg"
      }
    ]
  }'
```
</CodeGroup>

### Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `chain` | string | ✅ | The blockchain of the contract |
| `contractAddress` | string | ✅ | The contract address under which you want to create the token-type |
| `creations` | array | ✅ | An array of objects that can define one or multiple token-type details |
| `creations.name` | string | ✅ | The name of the token-type |
| `creations.description` | string | ❌ | The description of the token-type |
| `creations.image` | string | ❌ | The image URL for the token-type that will be displayed |

<Tip>
**Batch Creation:** You can create multiple token-types in a single request by adding more objects to the `creations` array.
</Tip>

## Token-Type Status

The token-type creation is asynchronous. Monitor the status:

| Status | Description |
|--------|-------------|
| `PENDING` | Token-type creation is in progress |
| `SUCCESS` | Token-type successfully created and ready for minting |
| `FAILED` | Creation failed - check error details |

## Advanced Token-Type Features

<AccordionGroup>
  <Accordion title="Adding Custom Attributes" icon="tags">
    **Enhance your token-types with custom attributes:**
    
    ```json
    {
      "creations": [
        {
          "name": "Legendary Sword",
          "description": "A powerful weapon",
          "image": "https://example.com/sword.jpg",
          "attributes": [
            {
              "type": "property",
              "name": "Rarity",
              "value": "Legendary"
            },
            {
              "type": "stat",
              "name": "Attack Power",
              "value": "95",
              "maxValue": "100"
            }
          ]
        }
      ]
    }
    ```
  </Accordion>
  
  <Accordion title="Fungible vs Non-Fungible" icon="coins">
    **Token-type behavior:**
    - **Non-fungible (default)**: Each mint gets a unique token ID
    - **Fungible**: All mints share the same token ID
    
    ```json
    {
      "creations": [
        {
          "name": "Game Currency",
          "fungible": true,
          "maxSupply": 1000000
        }
      ]
    }
    ```
  </Accordion>
  
  <Accordion title="Supply Limits" icon="chart-line">
    **Control token supply:**
    
    ```json
    {
      "creations": [
        {
          "name": "Limited Edition NFT",
          "maxSupply": 100,
          "description": "Only 100 will ever exist"
        }
      ]
    }
    ```
  </Accordion>
  
  <Accordion title="Media and Animation" icon="video">


## Best Practices

<Tip>
**Token-Type Optimization:**

1. **Descriptive Names** - Use clear, searchable names
2. **High-Quality Images** - Use at least 400x400px images
3. **Rich Metadata** - Add relevant attributes and properties
4. **Supply Planning** - Set appropriate max supply limits
5. **Batch Creation** - Create multiple token-types efficiently
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Invalid contract address">
    **Possible causes:**
    - Contract not deployed or still pending
    - Wrong blockchain specified
    - Incorrect contract address format
    
    **Solution:** Verify the contract is successfully deployed and the address is correct.
  </Accordion>
  
  <Accordion title="Image URL not accessible">
    **Requirements:**
    - URL must be publicly accessible
    - Use HTTPS protocol
    - Supported formats: JPG, PNG, GIF, SVG
    - Recommended size: 400x400px or larger
    
    **Solution:** Test the image URL in a browser to ensure it loads.
  </Accordion>
  
  <Accordion title="Token-type creation failed">
    **Common causes:**
    - Invalid metadata format
    - Network congestion
    - Insufficient permissions
    
    **Solution:** Check the creation status using the creation ID and retry if necessary.
  </Accordion>
</AccordionGroup>
