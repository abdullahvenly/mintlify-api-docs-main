Wallet API users can now create a password signing method to lock/unlock their wallets and retrieve the native token's USD price. We have also expanded response bodies for Wallet and NFT API endpoints.

## New `PASSWORD` signing method for your users

In addition to the `PIN`, `EMERGENCY_CODE`, and `BIOMETRIC` signing methods, you can now create a `PASSWORD` signing method as well. The password signing method can be used like others to unlock a wallet, execute transactions/signatures, update/recover another signing method, etc.

> ðŸ“˜ 
> 
> - The password can contain a combination of alphanumeric characters (_letters and numbers_) as well as special characters. 
> - Alphanumeric characters can be either uppercase or lowercase letters (A-Z, a-z) and digits (0-9), and special characters.
> - This allows for stronger and more complex passwords, enhancing security.

The following endpoint is used to create a password singing method for your users:

### Request Endpoint: [reference](https://docs.venly.io/reference/createsigningmethod)

```http
POST /api/users/{userId}/signing-methods
```

**Header Parameters:**

| Parameter        | Param Type | Value      | Description                                                                                     | Example Value                                     |
| :--------------- | :--------- | :--------- | :---------------------------------------------------------------------------------------------- | :------------------------------------------------ |
| `Signing-Method` | Header     | `id:value` | `id`: This is the ID of the signing method<br/>`value`: This is the value of the signing method | `756ae7a7-3713-43ee-9936-0dff50306488:123456` |

### Request Params & Request Body:

| Parameter  | Param Type | Description                                                                  | Data Type | Mandatory |
| :--------- | :--------- | :--------------------------------------------------------------------------- | :-------- | :-------- |
| `{userId}` | Path       | The ID of the user for whom to create the signing method for                 | String    | âœ…         |
| `type`     | Body       | The type of signing method to create for this user. In this case: `PASSWORD` | String    | âœ…         |
| `value`    | Body       | The password to create.                                                      | String    | âœ…         |

```json
{
    "type": "PASSWORD",
    "value": "QWERty123%^&*"
} 
```

### Response Body:

```json
{
    "success": true,
    "result": {
        "id": "4583c952-bd7d-4e3f-8426-be1b3ffb68d7",
        "type": "PASSWORD",
        "incorrectAttempts": 0,
        "remainingAttempts": 3,
        "hasMasterSecret": true
    }
}
```

## Fetch the native token's USD price

The following endpoints now return an `exchange` object in the response that includes the USD price of your native tokens:

1. Get all wallets: [`GET /api/wallets`](https://docs.venly.io/reference/viewwallets_1) where `includeBalance=true`
2. Get native Balance by wallet Id: [`GET /api/wallets/{walletId}/balance`](https://docs.venly.io/reference/viewbalance)
3. Get native Balance by Chain and Wallet Address: [`GET /api/wallets/{secretType}/{walletAddress}/balance`](https://docs.venly.io/reference/viewbalance_1)

### Example Response Body:

Following is an example response when querying the **Get native Balance by wallet Id** endpoint, where we get back an `exchange` object in the response.

| Param               | Description                                            |
| :------------------ | :----------------------------------------------------- |
| exchange.`usdPrice` | The USD price of one native token                      |
| exchange.`usdValue` | The cumulative USD value of your native token holdings |

```json
{
    "success": true,
    "result": {
        "available": true,
        "secretType": "ETHEREUM",
        "balance": 0.001,
        "gasBalance": 0.001,
        "symbol": "ETH",
        "gasSymbol": "ETH",
        "rawBalance": "1000000000000000",
        "rawGasBalance": "1000000000000000",
        "decimals": 18,
        "exchange": {
            "usdPrice": 3060.7453652388544,
            "usdBalanceValue": 3.060745365238854
        }
    }
}
```

## Retrieve the `tokenId` when you fetch the wallets that own NFTs from a specific contract

The following endpoints now return the `tokenId` param in the response that indicates the `id` of the token present in the wallet.

1. `GET /api/contracts/{secretType}/{address}/wallets`
2. `GET/api/nonfungibles/{secretType}/{contractAddress}/{tokenId}/wallets`
