Key highlights include the addition of parameters in the response bodies. A `storage` object in token-type responses, `burnable` and `supply` details, and expanding filtering capabilities for token mints and contracts. Additionally, we've introduced support for company-specific queries across multiple endpoints, enabling better asset organization and management.

## Token-API + NFT-API Version 2.0 and 3.0 Updates

### 1. Storage Object, Burnable, and Supply Object Added to Get Token-Type by ID Responses (V2 & V3)

**Endpoints Affected:**
- [`GET /api/v2/token-types/{id}`](https://docs.venly.io/v2.0/reference/gettokentype_1-1)
- [`GET /api/v3/erc1155/token-types/{id}`](https://docs.venly.io/reference/gettokentype_1)

**Change:** 
- Added a `storage` object to the response body, which includes `type` and `location` fields.
- Added `burnable` field to response body to indicate if the token is burnable.
- Added a `supply` object to the response body, which includes `current`, `pending`, and `max` fields that indicate the number of current mints, pending mints, and maximum mints for a token-type.

### 2. Filter All Token Mints by Destination Address (V2 & V3)

**Endpoints Affected:**
- [`GET /api/v2/tokens/mints`](https://docs.venly.io/v2.0/reference/getall-1)
- [`GET /api/v3/erc1155/tokens/mints`](https://docs.venly.io/reference/getall-1)

**Change:** 
- `destinationAddress` query param added.
- Added the ability to filter token mints by `destinationAddress`.

### 3. Filter All Token Mints by Contract Address (V2 & V3)

**Endpoints Affected:**
- [`GET /api/v2/tokens/mints`](https://docs.venly.io/v2.0/reference/getall-1)
- [`GET /api/v3/erc1155/tokens/mints`](https://docs.venly.io/reference/getall-1)

**Change:** 
- `contractAddress` query param added.
- Added the ability to filter token mints by `contractAddress`, supporting both complete and partial matches.

### 4. Supply Object Included in Get All Token-Types Responses (V2 & V3)

**Endpoint Affected:**
- [`GET /api/v2/token-types`](https://docs.venly.io/v2.0/reference/getalltokentypes)
- [`GET /api/v3/erc1155/token-types`](https://docs.venly.io/reference/getalltokentypes)

**Change:** 
- The `supply` object is now included in the response for get all token types.
- The `supply` object includes `current`, `pending`, and `max` fields that indicate the number of current mints, pending mints, and maximum mints for a token-type.

### 5. CompanyId Query Param for Get Contract Endpoints

**Endpoints Affected:**
- [`GET /api/v2/contracts`](https://docs.venly.io/v2.0/reference/getcontracts-1)
- [`GET /api/v3/erc1155/contracts`](https://docs.venly.io/reference/getcontracts)
- [`GET /api/v3/erc20/contracts`](https://docs.venly.io/reference/getallerc20contracts)

**Change:** 
- Added the ability to pass a `companyId` query parameter to filter contracts by company.

### 6. CompanyId Query Param for Get All Token Mints Endpoints (V2 & V3)

**Endpoints Affected:**
- [`GET /api/v2/tokens/mints`](https://docs.venly.io/v2.0/reference/getall-1)
- [`GET /api/v3/erc1155/tokens/mints`](https://docs.venly.io/reference/getall-1)

**Change:** 
- Added the ability to pass a `companyId` query parameter to filter mints by company.

### 7. CompanyId Field in Contract Deployment Requests

**Endpoints Affected:**
- [`POST /api/v2/contracts/deployments`](https://docs.venly.io/v2.0/reference/deploycontract-1)
- [`POST /api/v3/erc1155/contracts/deployments`](https://docs.venly.io/reference/deploycontract)
- [`POST /api/v3/erc20/contracts/deployments`](https://docs.venly.io/reference/deployerc20contract)

**Change:** 
- Added a `companyId` field to the request body for contract deployment, allowing the association of contracts with a specific company.

**Example Request Body** (_NFT V3_)

```json
{
    "name": "Venly-Test-Contract ",
    "symbol": "DNC",
    "description": "This NFT collection is created for testing.",
    "image": "https://storage-staging.venly.io/applications/bff93371-db40-4363-9692-4fbe64a34b54/logovenly.png",
    "externalUrl": "https://www.venly.io/",
    "chain": "MATIC",
    "companyId": "f407a5c1-12dc-4d5c-8fd3-71d7d462009f"
}
```
